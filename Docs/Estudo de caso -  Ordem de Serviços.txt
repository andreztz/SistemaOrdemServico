Estudo de caso Ordem de Serviço

Cliente: X (Assitência Técnica de microcomputadores, notebooks e perifericos.)

Necessidade do cliente: Sistema para controle de Ordem de Serviço.



Análise de Requisitos:

- Cadastro de clientes.
- Emissão de Ordem de Serviço.
- Relatório de Serviços.


Restrições:

Não pode existir nenhuma OS sem cliente vinculado.

O sistema deverá gerar automaticamente data e hora na emissão da OS.

Somente o gestor pode ter acesso ao relatório de serviços.
